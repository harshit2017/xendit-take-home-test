# Food Delivery App - Take Home Test Requirements

## Overview
This take-home test is designed to evaluate a candidate's ability to understand an existing codebase, implement new features, and follow best practices in Node.js/TypeScript backend development. The test focuses on extending a food delivery application with additional functionality that would enhance the user experience and add business value.

## Base Application
The base application is a food delivery platform backend with the following core functionality:
- User authentication and authorization (customer, restaurant owner, delivery person, admin)
- Restaurant management (CRUD operations)
- Menu and food item management
- Order processing
- Delivery tracking
- Payment processing

## Test Requirements
Candidates will be required to implement new features that extend the existing functionality. The implementation should:
- Follow the established architecture patterns
- Maintain code quality and consistency
- Include appropriate tests
- Be well-documented
- Consider security, performance, and scalability

## Time Allocation
Candidates should spend approximately 12 hours on this test if planning to implement most asks. The focus should be on quality rather than quantity of features implemented.

## Evaluation Criteria
Candidates will be evaluated based on:
1. Code quality and organization
2. Understanding of the existing codebase
3. Implementation of required features
4. Test coverage
5. Documentation
6. Problem-solving approach
7. Attention to security and performance considerations
