# Food Delivery App - Test Instructions and Evaluation Criteria

## Instructions for Candidates

### Getting Started
1. Clone the repository to your local machine
2. Install dependencies with `npm install`
3. Create a `.env` file based on the provided `.env.example`
4. Start the development server with `npm run dev`
5. Review the existing codebase to understand the architecture and functionality
6. Choose one or more features to implement from the provided task list

### Implementation Guidelines
1. **Code Quality**: Maintain the existing code style and architecture patterns
2. **TypeScript**: Use proper typing throughout your implementation
3. **Testing**: Write unit tests for all new functionality
4. **Documentation**: Document your code and update API documentation as needed
5. **Git Workflow**: Make atomic commits with clear messages
6. **Feature Completeness**: Ensure your implementation meets all requirements

### Submission Requirements
1. Submit your solution as a Git repository (GitHub, GitLab, etc.)
2. Include a README.md file with:
   - Overview of implemented features
   - Setup instructions
   - API documentation for new endpoints
   - Explanation of design decisions
   - Any challenges faced and how you overcame them
   - Suggestions for future improvements
3. Ensure all tests pass with `npm test`
4. Include any necessary database migrations or seed data

### Time Management
- You have 4-6 hours to complete this test
- It's better to fully implement one feature well than to partially implement multiple features
- Focus on quality over quantity
- If you run out of time, document what you would have done with more time

## Evaluation Criteria

### Code Quality (30%)
- **Clean Code**: Well-organized, readable, and maintainable
- **Architecture**: Proper separation of concerns and adherence to established patterns
- **Error Handling**: Comprehensive error handling and validation
- **Performance**: Efficient implementation with consideration for scalability
- **Security**: Proper authentication, authorization, and data validation

### Feature Implementation (30%)
- **Completeness**: All requirements for the chosen feature(s) are implemented
- **Correctness**: Implementation works as expected without bugs
- **Edge Cases**: Consideration of edge cases and error scenarios
- **Integration**: Proper integration with existing functionality

### Testing (20%)
- **Coverage**: Comprehensive test coverage for new functionality
- **Quality**: Tests are meaningful and verify correct behavior
- **Organization**: Tests are well-organized and follow established patterns

### Documentation (10%)
- **Code Comments**: Clear and helpful comments where necessary
- **API Documentation**: Complete documentation for new endpoints
- **README**: Clear explanation of implementation and design decisions

### Problem-Solving Approach (10%)
- **Understanding**: Demonstrated understanding of the problem domain
- **Creativity**: Innovative solutions to complex problems
- **Trade-offs**: Thoughtful consideration of trade-offs in design decisions
- **Future Considerations**: Awareness of potential future requirements and extensibility

## Bonus Points
- Implementation of additional features beyond the minimum requirements
- Deployment instructions or containerization
- Performance optimizations with benchmarks
- Advanced testing strategies (integration tests, load tests, etc.)
- Thoughtful refactoring of existing code (with explanation)
